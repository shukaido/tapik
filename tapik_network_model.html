1. BNM 규제 — 통합 포인트 합법성 확인

결론: Tapik 네트워크 내 통합 포인트는 BNM 라이선스 불필요

2024년 BNM Limited Purpose Electronic Money Exemption Order [P.U.A 463/2024]에 의거:
"Limited purpose EMI"에는 "e-money used in a closed-loop network (e.g., a network of merchants)" 및 "cash rewards or loyalty points"가 포함되며, 이는 BNM 규제 대상에서 완전 면제.

Tapik 적용: Tapik 파트너 가맹점 네트워크 내에서만 사용 가능한 통합 포인트 = "closed-loop network of merchants" = 면제 대상.
선례: BonusLink (540만 회원, Shell·AmBank·Parkson 네트워크)이 동일 구조로 운영 중.

금지선: 포인트 → 현금 환전, Tapik 네트워크 외부 사용 시 면제 자격 상실 → 라이선스 필요 (자본금 RM 1M~5M).
구조 BNM 판정 Tapik 적용
단일 매장 포인트 (Starbucks Rewards) 면제 이전 설계 (폐기)
가맹점 네트워크 통합 포인트 면제 (closed-loop network) 캡틴 구상 = 이것
범용 e-wallet (TnG, GrabPay) 라이선스 필요 해당 없음

2. 수익 모델 v4 — 시간 절약 프리미엄 + 포인트 환원

모델 핵심: 소비자가 편의(선주문·즉시 픽업)에 대해 5% 서비스료를 지불하고, 그 5%가 포인트로 100% 환원된다. 가맹점은 10% 수수료(코어)를 Tapik에 지불한다. Payment Gateway를 통해 Tapik은 자금을 직접 만지지 않는다.

2.1 참여자와 인센티브 (v4)

참여자 역할 인센티브 비용 부담
소비자 앱으로 선주문 + 선결제. 매장에서 QR 픽업 대기시간 0. 5% 서비스료 전액 포인트 환원. 크로스 가맹점 포인트 사용 메뉴가 + 5% 서비스료 (포인트로 돌아옴)
가맹점 주문 접수 → 조리 → QR 확인 후 제공 추가 매출 (5% 프리미엄 포함). 서빙·좌석·설거지 비용 절감. 거래 없으면 비용 0 Tapik 수수료 10% 코어 (거래 발생 시에만)
Tapik (본사) 플랫폼 운영. 주문·결제·포인트 관리 가맹점 수수료 10%에서 포인트(5%)·게이트웨이(~1.2%) 차감 후 순수익 포인트 지급 비용 + 게이트웨이 수수료

2.2 거래 흐름 (v4 Transaction Flow)

Step 1 — 소비자가 앱에서 주문
소비자 앱 → 근처 가맹점 선택 → 메뉴 선택 → 주문 확인
나시르막 RM 10.00 + 즉시 준비 서비스료 5% RM 0.50 = 합계 RM 10.50
↓
Step 2 — 결제 (Payment Gateway)
FPX / DuitNow / TnG / GrabPay 선택 → 결제 승인
RM 10.50 → Payment Gateway 에스크로 계좌 도착. Tapik 계좌를 경유하지 않음.
↓
Step 3 — 가맹점 POS에 주문 전송
결제 완료 즉시 가맹점 POS/태블릿에 주문 티켓 자동 전송
가맹점 조리 시작. 예상 완료 시간 소비자 앱에 표시
↓
Step 4 — 픽업 & QR 검증
소비자 도착 → 앱 QR 코드 제시 → 가맹점 POS/태블릿으로 스캔
서버 검증: orderID + userID + timestamp 해시 → 1회 사용 후 무효화
↓
Step 5 — 정산 & 포인트 적립
Gateway가 자동 분배 (T+1~T+3일):
가맹점 정산: RM 10.50 - RM 1.05 (수수료 10%) = RM 9.45
Tapik 수수료: RM 1.05
Gateway 수수료: ~RM 0.28 (Tapik 몫에서 차감)

소비자 포인트: +50P (RM 0.50 상당) 즉시 적립
Tapik 순수익: RM 1.05 - RM 0.50 - RM 0.28 = RM 0.27~0.50 (Gateway별 상이)

2.3 수익 분배 구조 (v4)

항목 누가 부담 얼마 설명
5% 서비스료 소비자 메뉴가의 5% "즉시 준비 서비스료" 명목. 전액 포인트로 환원되므로 소비자 실질 부담 = 0
Tapik 수수료 가맹점 총 결제액의 10% (코어) 거래 발생 시에만. 월정액 없음. GrabFood 픽업 20% 대비 1/2 수준
포인트 비용 Tapik (수수료에서 차감) 서비스료 전액 (결제액의 ~4.76%) 소비자 5% 서비스료를 포인트로 변환. Tapik 수수료 수입에서 충당

3. Tapik 통합 포인트 경제 설계

3.1 포인트 발행 & 사용 규칙

항목 규칙
포인트 단위 1 Tapik Point = RM 0.10 가치. 100 TP = RM 10
적립 경로 ① 선주문 서비스료: 메뉴가 5% 서비스료 전액 포인트 환원 · RM 10.50 주문 → 50 TP (핵심 적립 경로)
② 추천 보상: 친구 첫 구매 시 양쪽 50 TP (1회성)
③ 미션/챌린지: 월간 "5곳 방문" 뱃지 50 TP 등 (v2)
④ 룸마스터 보너스: 그룹 룸 개설 시 추가 포인트 (v1.5)
사용 범위 Tapik 파트너 가맹점 어디서든 사용 가능 (통합 closed-loop). 가맹점 A에서 적립 → 가맹점 B에서 사용 OK.
사용 방식 결제 시 포인트 차감 (100 TP 이상부터). 선결제 시 포인트 부분 결합 가능.
유효 기간 적립일로부터 12개월. 3개월 미사용 시 알림. 만료 30일 전 재알림.
현금 환전 불가 (BNM 면제 조건 유지). 타인 양도도 불가.
포인트 정산 (가맹점) 소비자가 가맹점 B에서 포인트 사용 → Tapik이 가맹점 B에 포인트 상당액 정산. 포인트 비용은 Tapik이 부담 (수수료 수익으로 충당).

3.2 할인 이벤트 참여 시 포인트 되돌림 구조

캡틴 구상: "할인 이벤트 참여 시, 할인되는 만큼 포인트로 적립해주는 구조"

시나리오: Lim Kopitiam이 "점심 Lunch Set RM 14 → 10% 할인 RM 12.60" 이벤트 등록.

소비자 입장: RM 12.60 결제 + 14 Tapik 포인트 적립 (할인분 RM 1.40 상당)
→ 소비자는 할인도 받고 포인트도 얻음 → 더블 인센티브

비용 구조:
• 할인 RM 1.40 → 가맹점 부담 (자발적 프로모 비용)
• 포인트 14 TP (= RM 1.40 가치) → 누가 부담?

포인트 비용 부담 방안 3가지

안 구조 장점 단점 추천도
A 가맹점 부담
할인 비용 + 포인트 비용 모두 가맹점		 Tapik 재무 건전성 유지. 가맹점이 프로모 전체를 자기 비용으로 집행. 가맹점 부담 과중 (할인 10% + 포인트 10% = 실질 20% 비용). 가맹점이 거부할 가능성 높음. 비추
B Tapik 부담
할인은 가맹점, 포인트는 Tapik 수수료에서 차감		 가맹점 부담 최소화. 가맹점은 "할인"만 비용, Tapik이 "포인트 적립"이라는 부가 가치 제공. Tapik 수수료 수익이 포인트 지급에 잠식. 초기 적자 불가피 (수수료 5% 받고 포인트 10% 지급하면 역마진). 조건부
C 분담 구조 (추천)
할인: 가맹점 부담
포인트: 가맹점 "광고비" 명목으로 할인분의 50% 추가 부담 → Tapik이 포인트로 전환 지급		 가맹점은 "할인 + 광고비"로 이해 (배달앱 30% 대비 훨씬 저렴). Tapik은 광고비 수취로 포인트 재원 확보. 양측 부담 분산. 가맹점에게 "할인 비용 + 광고비"를 설명해야 하는 영업 복잡성. 그러나 GrabFood 30% 대비 "총 15%"라는 비교 논리로 극복 가능. 추천

안 C 상세 수치 예시

Lim Kopitiam 점심 이벤트 (안 C 적용)

정가: RM 14.00
할인 10%: -RM 1.40 (가맹점 부담)
소비자 결제: RM 12.60

가맹점 추가 "광고비": 할인분의 50% = RM 0.70
→ Tapik이 이 RM 0.70을 7 Tapik 포인트로 변환해 소비자에게 지급

Tapik 수수료: RM 12.60 × 5% = RM 0.63

최종 정산:
가맹점 수취: RM 12.60 - RM 0.63 (수수료) - RM 0.70 (광고비) = RM 11.27
가맹점 총 비용: RM 14.00 - RM 11.27 = RM 2.73 (정가의 19.5%)
→ GrabFood 30% 대비 10%p 이상 저렴

소비자 혜택: RM 1.40 할인 + 7 TP (≈ RM 0.70 가치) = 총 RM 2.10 혜택 (15%)

Tapik 수취: RM 0.63 (수수료) — 포인트 비용은 가맹점 광고비로 커버

8. 포인트 규제 & 리스크 관리 — 자체 규제 프레임워크

8.1 BNM 면제 상태에서의 자체 규제 필요성

핵심: P.U.A 463/2024에 의해 Tapik 포인트는 BNM 관할 외. BNM이 강제하는 상한선·이전 제한·준비금 규정이 없다.
이것은 "자유"가 아니라 "안전망도 없다"는 뜻.

BNM이 원래 Limited Purpose EMI에 설계했던 기준 (유저당 RM 250, 총 발행량 RM 100만)은 면제령으로 효력 상실.
그러나 Consumer Protection Act 1999 (Act 599) Part IIIA "불공정 약관" 조항은 여전히 적용.
따라서 Tapik이 자체 T&C(이용약관)로 규제 프레임워크를 구축해야 함.

8.2 유저당 포인트 잔액 상한

항목 결정
잔액 상한 10,000 TP = RM 1,000 (≈ KRW 310,000)
캡틴 판단: RM 250은 초기 부스팅에 너무 적음. RM 1,000이면 충분한 적립 여유 + 재무 통제 가능.
상한 도달 시: 신규 적립 중단 알림 → "사용 후 적립" 전환 또는 자동 쿠폰 변환 유도.
유동성 풀(Reserve) 포인트 대량 사용 시 가맹점이 아닌 Tapik 본사가 정산.
가맹점은 포인트 결제를 수락만 하고, Tapik이 해당 금액을 정산 주기에 맞춰 지급.
Tapik 본사에 미상환 포인트 총량 대비 준비금(Reserve Fund) 적립 필수.
초기 비율: 분기별 미상환 총량의 10% (차후 데이터 기반 조정).
포인트 만료(Rolling Expiry) BNM 관할 밖이므로 만료 기간 설정은 Tapik 재량. 규제 없음
마지막 활동일 기준 12개월 (Rolling). 활동 = 적립 또는 사용.
BonusLink는 36개월 고정 만료. Tapik은 활동 시 리셋이므로 CPA 불공정 약관 분쟁 소지 최소화.
만료 3개월 전 앱 알림 + 30일 전 재알림.

8.3 P2P 이전 금지 & 대체 선물 메커니즘

원칙: 비양도, 비매매
BonusLink 선례를 따름. 포인트는 타인에게 이전·판매·양도 불가. 사망 시 유족 이전은 Tapik 재량.

이유: P2P 이전이 가능해지면 포인트가 사실상의 결제/송금 수단 → P.U.A 463/2024 면제 범위 이탈 → BNM 라이선스 필요.
글로벌 데이터: P2P 이전 가능한 포인트 프로그램은 단일채널 대비 2.5배 더 사기 공격을 받음.

8.4 포인트 제도 허점 & 방어 전략

리스크 시나리오 방어책 T&C 반영
포인트 파밍
(Points Farming)		 가짜 계정 대량 생성 → 신규가입 보너스(50 TP) 수확 → 한 계정으로 몰기 시도 P2P 이전 차단으로 1차 방어. 같은 기기/전화번호 다중 계정 탐지. 신규 보너스는 첫 구매 완료 후 지급 (가입만으로 X). 필수
포인트 세탁
(Points Laundering)		 도난 카드로 구매 → 포인트 적립 → 포인트로 상품 교환 결제 취소/환불 시 포인트 즉시 회수. 차지백 발생 시 해당 계정 적립분 전액 동결. 이상 거래 탐지(비정상적 고액·고빈도 구매 패턴). 필수
가치 변동 분쟁
(Devaluation Risk)		 Tapik이 1 TP = RM 0.10 → RM 0.05로 갑자기 변경 T&C에 명시: 가치 변경 시 30일 사전 고지. 기존 적립분은 변경 후 6개월간 구가치 적용. CPA 1999 Part IIIA (불공정 약관) 위반 방지. 필수
만료 불공정 분쟁 소비자가 "기간 너무 짧다"며 CPA 불공정 약관으로 문제 제기 Rolling expiry (활동 시 리셋)이므로 "적극 사용자는 만료되지 않는" 구조. BonusLink 36개월 고정 대비 오히려 유리. 만료 전 충분한 알림 (3개월 + 30일 전). 방어됨
가맹점 이탈 시
포인트 보장		 가맹점 A 탈퇴 → A에서 적립한 포인트 사용 불가? 통합 포인트이므로 가맹점 탈퇴와 무관하게 다른 가맹점에서 사용 가능. T&C에 "포인트는 Tapik 네트워크 전체에서 유효" 명시. 특정 가맹점에 귀속되지 않음. 방어됨
크로스 가맹점 재정거래
(Arbitrage)		 가맹점 A의 높은 할인 이벤트에서 포인트 대량 적립 → 가맹점 B에서 전액 사용 포인트 적립률은 Tapik이 통제 (가맹점별 자율 설정 불가). 이벤트 포인트는 별도 "보너스 TP"로 분리, 사용 시 비율 제한 가능 (예: 결제액의 최대 30%까지 포인트 사용). 필수
미상환 포인트
재무 부담		 발행 총량이 쌓이면서 Tapik 재무제표상 부채 증가 미상환 포인트 준비금(Reserve Fund): 분기별 미상환 총량의 최소 10% 적립 (차후 조정). Rolling expiry로 자연 소멸 유도. 분기별 breakage rate(미사용 소멸 비율) 모니터링. 필수
