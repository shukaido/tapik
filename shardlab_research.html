과제 개요

미션: 글로벌 Web3 VC 산하 벤처 스튜디오(ShardLab)에 합류, 말레이시아 선도 POS 사업자와의 파트너십을 활용해 가맹점이 아닌 소비자에게 직접 가치를 제공하는 B2C 프로덕트를 기획·프로토타입.

제출물

  • 워킹 프로토타입 — 실제 인터랙티브, AI 툴로 빌드 (정적 목업 불가)
  • 프로덕트 브리프 6파트
    1. 기회 발굴 (시장 갭 + 경쟁)
    2. 타겟 유저 & 문제 정의
    3. 프로덕트 전략 & 우선순위
    4. 워킹 프로토타입 + AI 활용 과정 문서화
    5. 프로덕트 스펙 (개발자/디자이너 착수 가능 수준)
    6. 검증 & 실행 계획 (리스크 3 + 실험 + 12주 로드맵)

평가 관점

  • 프로덕트 사고 — 기회 발굴, 트레이드오프, 논리 구조
  • 양보다 깊이 — "4개 깊게 > 6개 얕게" 명시
  • AI 활용의 판단력·반복·비판적 평가
  • POS 네트워크 레버리지 (이게 없으면 과제 자체의 의미 없음)

3. TP (Tapik Points) 시스템

  • TP_CAP: 월 100,000 TP (= RM 1,000) — 월별 상한선
  • 12개월 FIFO 만료 — 12개월 후 자동 소멸
  • 서비스료 5% 전액 즉시 포인트 환원
  • 매장별 closed-loop (타점포 환금 불가)

4. Room 6단계 상태 머신

    ↓

    ↓

    ↓

5. Room 타입

6. Creator 方爆 (방폭) 규칙

Q1. 배달 플랫폼이 우리 앱과 무슨 상관? 매장 vs 배달 비중? POS 통합 상태?

  • 직접 경쟁자 아님. 하지만 가맹점 운영비의 최대 축(수수료 25~35%)이라 우리 제품 세일즈 논리와 직결
  • 매장 dine-in 64%가 여전히 지배적. delivery는 F&B 시장의 18~20% 추정. 직장인 점심은 매장·takeaway 중심
  • 배달 주문은 선결제 완료 상태로 POS에 도달. 라이더 단말기 없음 (COD는 희소). 가맹점 POS에선 "GrabFood 주문 RM 35 접수" 형태로 들어옴
  • 통합 상태: StoreHub·Eats365·Qashier는 API 통합 제공. IRS 등 구형 POS는 수동 입력 중심 → 독립 가게 다수 수동
  • 캡틴 가설 확인: "원시 데이터는 있으나 시각화 안 됨" — 부분적 사실. API 통합된 POS만 데이터 수집 가능, 마진 계산·플랫폼 간 비교·광고 ROI 측정 도구 부재. 이게 B2B 기회
  • 단 이 축은 B2B 성격 강함 → 과제 B2C 제약과 충돌 → 가맹점 분석 결과를 소비자 혜택으로 되돌리는 구조 필요 (섹션 7 재평가)

Q2. BonusLink가 독립 가게 왜 거절? 단골 경제 근거 구체 수치?

  • 거절 이유 4가지: (1) 최소 파트너 조건 브랜드 중심 (2) 기술 통합 비용 (3) 포인트 발행 비용 모델 (매출의 0.5~1% 부담) (4) 가성비 — BonusLink 입장에서 독립 가게 추가해도 소비자 혜택 체감 증가 미미
  • 단골 경제 수치 풀이: 비단골 연 RM 300/명 → 단골 연 RM 2,016/명 (방문 4배 · 객단가 1.67배). 단골 100명 확보 시 연 RM 201,600 추가 매출

Q3. 할랄 투명성 — 무인증이 불법인데 어떻게 영업? 진짜 문제?
