F-1-2. 독립 가맹점이 쿠폰을 안 쓰는 5가지 이유

# 이유 심각도 상세
1 마진이 없다 매우 높음 순이익 5~12%에서 10% 할인을 주면 적자. "할인할 돈이 없다"가 가장 큰 이유
2 시스템이 없다 높음 POS 시스템 없이 계산기·종이 영수증 운영. 디지털 쿠폰 발행·추적·정산 인프라 자체가 부재. SME의 57%가 디지털 전환 미시작
3 필요성을 못 느낀다 중간 Mamak·Kopitiam은 고정 단골이 매출의 60~80%. "할인 안 해도 올 사람은 온다"는 인식
4 쿠폰 손님은 재방문 안 한다 중간 가맹점 사장들의 경험적 인식: "쿠폰으로 온 손님은 쿠폰 없으면 다시 안 온다." Cherry picker 문제
5 위 4가지를 해결해줄 파트너가 없다 높음 Fave는 도시 카페 위주. Mamak·Kopitiam을 찾아가서 교육·온보딩해주는 플랫폼은 현재 없음
⚠️ 캡틴의 직감이 맞다: 독립 F&B는 구조적으로 쿠폰을 쓰기 어렵다. "인기 있는 곳은 필요 없고, 어려운 곳은 여력이 없다"는 정확한 진단. Tapik의 "할인 + 포인트" 모델이 독립 F&B에 맞는지 재검토가 필요한 지점.

# 장벽 상세
1 현금 경제 투명화 공포 많은 독립 F&B가 현금 위주 운영. e-Invoice는 모든 거래를 정부에 실시간 보고. 기존에 신고하지 않던 현금 매출이 드러남 → 세금 부담 증가 우려. 이것이 가장 큰 심리적 저항

3 영업 중단 공포 시스템 전환 중 영업을 멈출 수 없음. "배우면서 장사하라"는 건 현실적으로 어렵다. 코로나 이후 아직 회복 안 된 곳 다수

가맹점이 가장 원하는 지원 (설문):
· 70%: "단계별 가이드·체크리스트" — 뭘 해야 하는지 쉽게 알려달라
· 59%: "기존 시스템 연동 도움" — 내가 쓰는 것과 연결해달라
· 53%: "솔루션 추천" — 뭘 사야 하는지 알려달라
→ 기술이 아니라 "안내"가 부족하다. 솔루션보다 가이드가 더 절실하다

# 옵션 적합도 근거
A e-Invoice를 핵심 온보딩 훅으로 낮음 Tapik 주 타겟(Kopitiam·Mamak)은 면제 대상. "e-Invoice 도와준다"가 세일즈 포인트 안 됨. 의무 대상(RM 100만+)은 이미 StoreHub·AutoCount 등 기존 솔루션이 장악
B e-Invoice를 부가 기능으로 (있으면 좋은 것) 중간 Tapik POS에 e-Invoice 포함하되, 이것이 메인 세일즈 포인트가 아님. "성장하면 자동으로 e-Invoice 준비 완료"라는 미래 가치 제안. 가맹점이 RM 100만 넘기면 자연스럽게 전환
C e-Invoice 포기. 다른 훅 찾기 높음 면제 대상이 대부분인 시장에서 e-Invoice에 투자하는 건 비효율. F&B 페인포인트 1위(인력 부족), 3위(고객 확보)에 집중. e-Invoice는 시장이 성숙하면 추가
D 파트너십: JomeInvoice 등과 연동 중간 직접 구축 대신 미들웨어 파트너 API 연동 (4~6주). RM 100만+ 가맹점에만 제공. 최소 리소스로 "있으면 좋은" 기능 확보
🔍 최종 전략 판단 (캡틴 결정 필요):

1. e-Invoice는 Tapik의 "킬러 온보딩 훅"이 될 수 없다
   근거: 타겟 시장 대부분이 면제. 의무 대상은 기존 솔루션이 이미 서비스 중

2. 그러면 Tapik의 진짜 온보딩 훅은 뭐가 돼야 하나?
   F-2 페인포인트 순위 재확인:
   · 1위: 인력 부족 → Tapik이 해결할 수 없음 (플랫폼의 영역 아님)
   · 2위: 원가 상승 → Tapik이 해결할 수 없음 (공급망 문제)
   · 3위: 신규 고객 확보 → 이것이 Tapik의 유일한 현실적 진입점
   · "가맹점에 새 손님을 보내준다"가 가장 강한 세일즈 포인트

3. 옵션 B vs C vs D 중 선택 필요:

H-1. 말레이시아 결제 인프라 — Payment Gateway 모델

캡틴 질문: "선결제를 매장 안 가고, 우리 앱 내에서 하되, 우리가 돈을 직접 안 받고도 가능한가?"

결론: 가능. Payment Gateway 모델로 Tapik은 돈을 직접 만지지 않는다.
소비자 은행 → Payment Gateway → 가맹점 은행. Tapik은 수수료만 자동 차감 후 정산 지시.
BNM 금융 라이선스 불필요.

H-1-1. 결제 흐름

소비자가 앱에서 RM 10.50 결제 시:

① 소비자 → FPX/DuitNow/e-wallet 선택 → 결제 승인
② 결제금 RM 10.50 → Payment Gateway 에스크로 계좌 도착
③ Gateway가 자동 분배:
   · 가맹점 정산: RM 10.50 - RM 1.05 (Tapik 수수료 10%) = RM 9.45
   · Tapik 수수료: RM 1.05
   · Gateway 수수료: ~RM 0.13 (1.2%) + RM 0.30 (고정비) ← Tapik 몫에서 차감
④ T+1~T+3일 내 가맹점 은행 계좌로 입금
핵심: Tapik 회사 계좌를 돈이 경유하지 않음. Gateway가 직접 분배.

H-3-2. 가맹점 설득 논리 — "왜 10% (코어)를 내야 하나"

논리 내용
1. 5% 추가 매출을 만들어줌 소비자가 매장가보다 5% 더 지불함. 이 5%는 Tapik이 만들어낸 추가 수익. "우리가 5% 더 받아다 드렸으니 10% 달라"
2. GrabFood 픽업의 1/2 수준 GrabFood 픽업 20% vs Tapik 10% (코어). 가맹점 실수령: GrabFood RM 9.60 vs Tapik RM 11.34 (RM 12 기준) — Tapik이 더 유리
3. 숨은 비용 절감 픽업 주문은 서빙비(RM 0.80~1.20) + 자리비(RM 0.50~1.00) + 설거지(RM 0.20~0.40) 불필요. 실질 마진 매장 판매 대비 동등 또는 우위 (E-8 분석 참조)
4. 거래 없으면 비용 0 월정액 아님. Tapik 통해 주문 들어올 때만 수수료 발생. 리스크 제로
5. 룸 기능으로 바이럴 성장 v1 MVP에 룸 포함. 포인트 차등(Solo 5%, Joiner 6%, Creator 8%)으로 자연스러운 추천 인센티브. 가맹점을 위해 구매자 증가 추진 중

H-3-3. 소비자 수용성 — "왜 5% 더 내야 하나"

논리 내용
1. 배달앱 선례 GrabFood에서 매장가 대비 15~20% 비싸도 MAU 1,500만+. 소비자는 "앱 가격이 좀 비싼 건 원래 그런 것"으로 학습됨
2. 배달앱의 1/3 프리미엄 GrabFood 15~20% vs Tapik 5%. 그마저도 포인트로 환원. 실질 추가 부담 = 0
3. 명확한 명목 "즉시 준비 서비스료" — 줄 안 서고, 주문 후 바로 받음. 배달앱의 "배달비"처럼 직관적 명분
4. 포인트 전액 환원 5% 서비스료가 100% 포인트로 돌아옴. "돈을 더 쓴 게 아니라 적립한 것" 프레이밍

H-3-4. 가격 프레이밍 전략 — GrabFood 방식 채택

캡틴 확정: GrabFood 방식으로 간다

GrabFood에서 매장보다 비싼 걸 소비자들이 알지만 "원래 그런 거"로 받아들인다.

# 가정 리스크 검증 방법
