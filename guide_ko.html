Tapik 인터랙티브 데모 가이드

이 가이드를 따라가면서 Consumer와 Merchant 앱의 전체 주문 흐름을 실시간으로 체험해 보세요.

시작하기 전에

⚠ 중요: 두 탭 설정 필수

Tapik의 Consumer와 Merchant 앱은 BroadcastChannel API를 통해 실시간으로 통신합니다. 주문이 머천트 측에 표시되고, 상태 변경이 소비자 측에 즉시 반영되는 전체 경험을 보려면 두 앱을 같은 브라우저에서 열어야 합니다.

1
Consumer 앱 열기

Consumer 앱을 새 탭에서 여세요. 이것이 고객의 모바일 경험을 보여주는 메인 뷰입니다.

2
Merchant 앱 열기 (나란히 배치)

Merchant 앱을 다른 탭에서 여세요. 두 탭을 나란히 배치하세요 (Windows: Win+← / Win+→, Mac: 화면 가장자리로 드래그). Merchant 앱은 가맹점이 실제로 사용할 Lenovo Tab M7 태블릿의 가로 모드를 시뮬레이션합니다.

3
연결 확인

Merchant 앱의 상단 영역에 초록색 인디케이터가 "Consumer Online"으로 표시되는지 확인하세요. BroadcastChannel이 작동 중이고 두 앱이 통신 가능한 상태임을 의미합니다.

💡 체험할 내용

Consumer 앱에서 주문하면 Merchant 앱의 주문 대기열에 즉시 나타납니다. 머천트가 수락, 준비, 완료를 누르면 — Consumer 앱이 실시간으로 업데이트됩니다. 페이지 새로고침 없이요. 이것이 Tapik의 핵심입니다: 고객과 주방을 직접 연결하는 것, 중간자 없이.

Consumer 앱 — 단계별 워크스루

Consumer 앱은 폰 프레임 시뮬레이션 안에 모바일 퍼스트로 구현되어 있습니다. 로그인부터 주문 픽업까지 모든 핵심 플로우가 Firebase 백엔드와 완전히 연동됩니다.

Step 1: Google 로그인

1
"Start with Google" 탭

Firebase Authentication + Google Sign-In을 사용합니다. 버튼을 탭하고 Google 계정을 선택하면 바로 로그인됩니다. 프로필(이름, 사진)은 Google 계정에서 자동으로 가져옵니다.

LIVE Firebase Auth — 실제 인증. 계정이 Firestore에 생성됩니다.

Step 2: 홈 화면 둘러보기

2
홈 탭 둘러보기

로그인 후 홈 탭에 도착합니다. 여기서 볼 수 있는 것:

근처 매장 — F&B 가맹점 목록. 각 카드에 매장명, 업종, 거리, 평점, 실시간 대기열 상태(예: "3명 앞 · 약 9분")가 표시됩니다. 대기 인원은 Firestore의 실제 주문 데이터를 기반으로 실시간 계산됩니다.

"Right Now" 카드 — 실시간 딜을 보여주는 가로 캐러셀: 대기열 알림, 스팟 할인(비수기 2배 포인트), 핫딜(클리어런스 3배 포인트).

친구 소식 — 친구들이 어디서 먹고 있는지 표시. 목 데이터

Step 3: 매장 진입 & 메뉴 탐색

3
아무 매장 카드를 탭하세요 (예: "Feeka Coffee Roasters")

매장 상세 화면이 열리며 다음을 확인할 수 있습니다:

매장 정보 — 영업 상태(영업 중/마감), 평점, 거리, 현재 대기열. 커버 사진 영역에 찜/공유 버튼.

메뉴 아이템 — 각 아이템에 이름, 설명, 가격(RM), 추가(+) 버튼이 표시됩니다. 스크롤 가능한 단일 리스트로 구성.

"Room" 버튼 — Room 생성 플로우를 시작합니다 (Step 8에서 다룹니다).

👉 해보세요: 2~3개 아이템을 장바구니에 추가하세요. 하단에 아이템 수와 합계를 보여주는 플로팅 카트 바가 나타납니다.

Step 4: 장바구니 & 결제

4
하단의 카트 바를 탭하세요

결제 화면에서 주문 요약을 확인합니다:

아이템 목록 — 각 아이템의 수량 조절(−/+)과 금액.

소계 — 메뉴 가격 기준 전체 합계.

서비스료 (5%) — Tapik 수익 모델의 핵심. 소비자에게 5% 서비스료를 부과하고, 그 100%를 로열티 포인트(TP)로 환원합니다.

적립 예정 포인트 — 역할에 따라 동적 계산: Solo 주문 = 5% 적립, Room Joiner = 6%, Room Creator = 8% (Hotspot 모드에서 최대 10%).

결제 수단 — Touch 'n Go eWallet이 표시됩니다. UI만 구현 실제 결제 처리는 없습니다.

👉 해보세요: "결제 & 선주문"을 탭해서 주문을 넣어보세요.

Step 5: 주문 추적 (선결제 상태)

5
주문이 접수되었습니다

결제 후 자동으로 주문 탭으로 이동합니다. 주문이 "선결제" 카드로 표시되며 두 가지 옵션이 있습니다:

"지금 주문" — 주문을 머천트에게 즉시 전송합니다. 주문 상태가 ORDERED로 변경되고 Merchant 앱의 대기열에 나타납니다.

"나중에 보내기" — 선결제 주문을 최대 24시간 보관합니다. 준비되면 전송할 수 있습니다 (예: 오전 11시에 결제, 낮 12시에 주방으로 전송).

👉 해보세요: "지금 주문"을 탭하고 즉시 Merchant 앱 탭을 확인하세요.

실시간 동기화 1~2초 내에 Merchant 앱의 "New" 칼럼에 주문이 나타나야 합니다.

Step 6: Merchant 앱으로 전환 (주문 처리)

🔄 이제 Merchant 앱 탭으로 전환하세요

여기서부터가 핵심입니다. 머천트 입장에서 주문을 처리하면서, Consumer 앱이 실시간으로 업데이트되는 것을 확인하세요.

6a
주문 수락 (Merchant 앱)

Merchant 앱의 Orders 탭에서 "New" 칼럼에 새 주문이 보입니다. "Accept"를 탭하세요. 주문이 "Preparing" 칼럼으로 이동하며 준비 체크리스트가 표시됩니다.

👉 Consumer 앱을 확인하세요: 주문 상태가 "ORDERED"에서 "PREPARING"으로 애니메이션과 함께 변경됩니다.

6b
준비 완료 (Merchant 앱)

"준비가 완료"되면 Merchant 앱에서 "Ready"를 탭하세요. 주문이 "Awaiting Pickup" 칼럼으로 이동합니다.

👉 Consumer 앱을 확인하세요: 상태가 "READY"로 변경되며 — 픽업 검증용 QR 코드 그리드가 나타납니다.

Step 7: 픽업 & 포인트 적립

7
픽업 완료 (Merchant 앱)

Merchant 앱에서 "QR Pickup"을 탭하세요 (고객의 QR 코드 스캔을 시뮬레이션). 주문 상태가 PICKED_UP으로 변경됩니다.

👉 Consumer 앱을 확인하세요:

• 주문이 "PICKED UP ✓"로 표시되며 축하 애니메이션이 재생됩니다.

• 포인트(TP)가 자동으로 적립됩니다 — 리워드 탭에서 업데이트된 잔액을 확인하세요.

• 리뷰 프롬프트가 나타나며 — 별점과 코멘트를 남길 수 있습니다.

LIVE 포인트가 Firestore에 실시간으로 기록됩니다. 주문부터 픽업, 포인트 적립까지 — 전체 플로우가 완전히 작동합니다.

✅ Tapik의 전체 주문 사이클을 완료했습니다

소비자 선결제 → 머천트 수신 → 준비 → 완료 → 픽업 → 포인트 적립. 모두 두 브라우저 탭에서 Firebase Firestore 기반으로 실시간 동기화됩니다. 페이지 새로고침 없이, 폴링 없이 — 순수한 실시간 아키텍처.

Step 8: Room — 그룹 주문

8
Room 생성 또는 참여

Consumer 앱의 Room 탭으로 이동하세요. Room은 Tapik의 소셜 주문 기능입니다 — 여러 사람이 같은 매장에서 함께 주문하고 모두 부스트된 포인트를 적립합니다.

Room 생성:

• 매장을 선택하고, Room 유형(공개 또는 비밀번호 비공개)과 멤버 제한(2~8명)을 설정합니다.

• 생성된 Room은 라이브 Room 목록에 표시됩니다. 다른 사용자가 공개 Room을 보고 참여할 수 있습니다.

• Creator로서 주문 시작("주문 활성화")을 제어하고 Room을 종료할 수 있습니다.

Room 상태 머신:

대기 중 → 준비 완료 (최소 인원 충족) → 주문 중 → 전원 결제 완료 → 종료 중 → 삭제됨

포인트 부스트: Creator는 8% 적립 (Solo 5% 대비), Joiner는 6% 적립. Hotspot 모드에서 Creator는 최대 10%까지 적립 가능.

LIVE 완전한 Firestore CRUD — Room의 생성, 참여, 종료가 실시간으로 동작합니다. 최소 인원 미달 시 30분 후 자동 만료.

Step 9: 리워드 & 포인트

9
포인트 확인

리워드 탭에서 확인할 수 있는 것:

총 TP 잔액 — 누적 로열티 포인트. TP Cap(100,000 TP = RM 1,000 상한)까지의 진행 바가 표시됩니다.

레벨 시스템 — L1 Newbie → L2 Regular (500+ TP) → L3 Local (5,000+ TP). 레벨에 따라 향후 기능 접근 권한이 결정됩니다.

매장별 분류 — 각 매장에서 적립한 포인트를 표시합니다. LIVE

퀘스트 — 달성형 미션 (3개 매장 방문, 새 카페 시도, Room 참여 등). 목 데이터

리워드 샵 — 교환 가능한 리워드 카탈로그 (무료 커피, 쿠폰, 카야 토스트). 목 데이터

Step 10: 언어 전환 & 설정

10
언어 전환

내 정보 탭에서 언어 토글(EN/KO)을 탭하세요. 전체 앱이 영어와 한국어 사이를 즉시 전환합니다 — 500개 이상의 번역 키가 모든 UI 텍스트를 커버합니다.

실시간 동기화 언어 변경이 BroadcastChannel을 통해 Merchant, Admin 앱으로 전파됩니다 — 세 앱이 동시에 언어를 전환합니다.

Merchant 앱 — 개요

📋 Merchant 앱 참고 사항

Merchant 앱의 핵심 주문 관리 플로우는 완전히 작동합니다 — 주문 수신, 수락, 준비, 완료 모두 Firebase와 BroadcastChannel로 동작합니다. 다만, 분석 대시보드, 고객 데이터, 비즈니스 설정은 현재 목/플레이스홀더 데이터를 표시하며, 향후 개발 단계에서 구현될 예정입니다. 아래 설명에서 라이브 vs 목 기능을 표시해 두었습니다.

Merchant 앱은 Lenovo Tab M7 태블릿 프레임(가로 모드, 1024×600px) 안에 렌더링됩니다 — 가맹점이 실제로 매장 내에서 사용할 기기를 시뮬레이션합니다.

Admin 앱 — 개요

📋 Admin 앱 참고 사항

Admin 앱은 Tapik의 내부 운영 대시보드입니다. 주로 목/플레이스홀더 데이터로 구성되어 의도한 분석 및 관리 기능을 보여줍니다. 라이브 주문 피드와 보상 모드 설정만 Firebase와 완전히 연동되는 기능입니다. 나머지 지표, 차트, 테이블은 대표적인 샘플 데이터를 표시합니다.

Admin 앱은 Tapik 운영팀을 위한 다크 테마 대시보드 레이아웃을 사용합니다.

탭별 기능 안내

탭 목적 확인 사항 상태
Overview 플랫폼 전체 KPI (GMV, MRR, WAU, Room 전환율), 코호트 리텐션 히트맵, AI 인사이트, 라이브 주문 피드. Live Feed 섹션에서 실제 주문이 흐르는 것을 확인하세요 — Consumer에서 주문하면 여기에 나타납니다. LIVE 피드 KPI MOCK
Merchants 가맹점 테이블 — 7개 샘플 매장, 건강도 점수(색상 코드), 수수료 티어(10~12%), 월간 GMV, 수락률, 준비 시간. Tapik이 가맹점 성과를 어떻게 모니터링하고 위험 매장을 식별하는지 둘러보세요. MOCK

Room Room 분석 — 총 Room 수, 성공률, 평균 멤버, Room당 GMV. Room 이력 테이블. Room 지표를 확인해서 그룹 주문 기능의 비즈니스 임팩트를 파악하세요. MOCK
Settlement 주간 정산 요약 — GMV, 수수료, 발행 포인트, 게이트웨이 수수료, 순 매출, 가맹점별 정산 내역. Tapik의 3.5% 단위 경제학(10% 수수료 − 5% 포인트 − 1.5% 게이트웨이)이 실제 정산에 어떻게 반영되는지 확인하세요. MOCK
Anomalies 알림 시스템 — 주문 없음, 이탈 위험, 긴 준비 시간, 비정상 패턴 플래그. 알림 샘플을 확인해서 운영 모니터링 의도를 이해하세요. MOCK
Settings 보상 모드 선택기 — Default(5%), Spot Deal(2배 포인트, 비수기), Hotspot(3배 포인트, 한정 시간) 전환. 👉 해보세요: 여기서 보상 모드를 변경하고 Consumer 앱을 확인하세요 — 결제 화면의 포인트 계산이 실시간으로 새 모드를 반영합니다. LIVE

기술 요약

구성요소 기술 상세
아키텍처 단일 파일 HTML + React 18 + Babel 빌드 과정 없음. 각 앱이 하나의 .html 파일. 브라우저에서 열기 = 앱 실행.
백엔드 Firebase Firestore (서버리스) 실시간 NoSQL 데이터베이스. 모든 주문 데이터, 사용자 프로필, Room 상태, 보상 설정이 여기에 저장됩니다.
인증 Firebase Auth (Google Sign-In) 원탭 로그인. 첫 로그인 시 사용자 프로필이 Firestore에 동기화됩니다.
앱 간 동기화 BroadcastChannel API 브라우저 네이티브. Consumer ↔ Merchant ↔ Admin이 서버 왕복 없이 주문 이벤트와 언어 변경을 통신합니다.
다국어 커스텀 i18n (601개 키) 영어 + 한국어. 언어가 localStorage에 저장되고 모든 앱에서 동기화됩니다.
